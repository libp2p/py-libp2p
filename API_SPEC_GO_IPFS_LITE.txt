py-ipfs-lite API specification (go-ipfs-lite parity target)
============================================================

Purpose
- Define the first public API surface for py-ipfs-lite so the abstraction layer is aligned with go-ipfs-lite.
- Keep the API minimal and embeddable: DAGService-like methods + UnixFS file add/get + setup helpers.

Parity goals
- Mirror the go-ipfs-lite concepts: Peer, Config, AddParams, SetupLibp2p, Bootstrap, Add/Get/Remove, AddFile/GetFile.
- Keep implementations swappable (host/routing/datastore/blockstore/exchange separated by interfaces).
- Prefer Pythonic snake_case, but optionally expose Go-style compatibility aliases.

Public package exports (MVP)
- Peer
- Config
- AddParams
- setup_libp2p(...)
- default_bootstrap_peers()
- new_in_memory_datastore()
- Libp2pOptionsExtra (or equivalent default option set)


1) Config API
-------------
@dataclass(slots=True)
class Config:
    offline: bool = False
    reprovide_interval_seconds: int = 43200  # 12h default
    uncached_blockstore: bool = False
    bitswap_broadcast_max_random_peers: int = 64
    bitswap_broadcast_control_send_to_pending_peers: bool = False

Notes
- These map directly to go-ipfs-lite config behavior.
- Treat negative reprovide interval as "disabled reprovider".


2) AddParams API (for add_file)
-------------------------------
@dataclass(slots=True)
class AddParams:
    layout: str = "balanced"                # balanced | trickle
    chunker: str = "size-262144"            # same shape as go-ipfs-lite chunker string
    raw_leaves: bool = True
    hidden: bool = False
    shard: bool = False
    no_copy: bool = False
    hash_fun: str = "sha2-256"
    max_links: int = 174

Notes
- Keep defaults aligned with current IPFS/UnixFS expectations used in go-ipfs-lite.


3) Setup helper APIs
--------------------
def default_bootstrap_peers() -> list[PeerInfo]

def new_in_memory_datastore() -> Datastore

async def setup_libp2p(
    *,
    host_key: PrivateKey,
    secret: bytes | None,
    listen_addrs: list[Multiaddr],
    datastore: Datastore | None,
    extra_options: list[Libp2pOption] | None = None,
) -> tuple[Host, Routing]

Behavior targets
- setup_libp2p returns a ready host + dht/routing object (same role as go SetupLibp2p).
- if datastore is None, routing provider state is in-memory.


4) Peer constructor + lifecycle
-------------------------------
class Peer:
    @classmethod
    async def new(
        cls,
        *,
        datastore: Datastore,
        blockstore: BlockStore | None,
        host: Host | None,
        routing: Routing | None,
        config: Config | None = None,
    ) -> "Peer"

    async def close(self) -> None
    async def bootstrap(self, peers: list[PeerInfo]) -> None
    def session(self, ctx: Context | None = None) -> NodeGetter

Behavior targets
- `new` composes blockstore, exchange (offline/online), dag service, and reprovider.
- bootstrap is best-effort: attempts peer connects then bootstraps routing layer.


5) DAGService parity methods
----------------------------
class Peer:
    async def add(self, node: IPLDNode) -> CID
    async def get(self, cid: CID) -> IPLDNode
    async def remove(self, cid: CID) -> None

Behavior targets
- Add/Get/Remove should be the core node-level API.
- remove is local blockstore remove (same semantics as go-ipfs-lite).


6) UnixFS/file methods
----------------------
class Peer:
    async def add_file(
        self,
        source: BinaryIO | AsyncBinaryIO,
        params: AddParams | None = None,
    ) -> CID

    async def get_file(self, cid: CID) -> ReadSeekCloser

Behavior targets
- add_file performs chunking + DAG construction and returns root CID.
- get_file resolves CID and returns file-like reader abstraction (or equivalent wrapper).


7) Blockstore / exchange accessors
----------------------------------
class Peer:
    def block_store(self) -> BlockStore
    async def has_block(self, cid: CID) -> bool
    def exchange(self) -> Exchange
    def block_service(self) -> BlockService

Behavior targets
- expose internals similarly to go-ipfs-lite for embedding and advanced integrations.


8) Abstraction layer interfaces (internal contracts)
----------------------------------------------------
Datastore
- get/put/delete/query/close primitives.

BlockStore
- put/get/has/delete/get_size/all_keys.

Exchange
- get_block/get_blocks/notify_new_blocks/start/stop.

DagService
- add/get/remove/get_many.

Routing
- bootstrap/find_providers/provide/get_value/put_value (as needed for MVP).

Host
- id/addrs/connect/disconnect/open_stream/set_stream_handler/close.

Reason
- These interfaces isolate py-ipfs-lite from concrete py-libp2p wiring details.


9) Optional Go-compat naming aliases
------------------------------------
For developer familiarity, optionally expose aliases:
- SetupLibp2p -> setup_libp2p
- DefaultBootstrapPeers -> default_bootstrap_peers
- NewInMemoryDatastore -> new_in_memory_datastore
- AddFile -> add_file
- GetFile -> get_file
- BlockStore -> block_store
- HasBlock -> has_block

Primary docs should still recommend snake_case APIs.


10) MVP acceptance checklist for this API
-----------------------------------------
- Two peers can bootstrap and exchange a DAG node (add/get/remove flow works).
- add_file/get_file roundtrip works for small + chunked files.
- public constructor path supports embedding external host/routing/datastore.
- API remains stable without exposing internal bitswap example modules directly.
