Phased roadmap for py-ipfs-lite

1. Phase 0 — API contract + scope freeze
◦  Define Python API parity with go-ipfs-lite (Peer, setup helpers, add/get/remove, add_file/get_file).
◦  Decide MVP scope vs deferred features.
2. Phase 1 — Project bootstrap (py-libp2p style)
◦  Create pyproject.toml, package layout, tests layout, lint/type/test configs, pre-commit, optional Makefile.
◦  Ensure install + test + build workflows work from fresh clone.
3. Phase 2 — Core architecture + lifecycle
◦  Add core abstractions: config, peer lifecycle, storage adapters, service composition.
◦  Ensure clean startup/shutdown and error handling.
4. Phase 3 — Setup helpers (SetupLibp2p-style)
◦  Integrate with the external local py-libp2p repository to use real `Host`, `Routing`, and connection components rather than mocks.
◦  Implement helper constructors for host identity, listen addrs, DHT wiring, bootstrap peers.
◦  Make embedding simple while keeping lower-level control available.
5. Phase 4 — DAG + file operations (functional core)
◦  Lift reusable logic from your bitswap functions into library APIs.
◦  Implement stable methods for DAG block ops and file add/get roundtrips.
6. Phase 5 — Example split (library-first)
◦  Refactor monolithic bitswap example into thin demos that consume py_ipfs_lite public APIs.
◦  Keep implementation in library, not examples.
7. Phase 6 — interop/ foundation (py ↔ go)
◦  Create interop/ structure + scripts/containers for deterministic interop runs.
◦  Add first smoke tests: py -> go and go -> py file transfer + CID checks.
8. Phase 7 — Interop hardening
◦  Expand compatibility vectors: chunk boundary cases, raw vs DAG-PB behavior, timeout/retry behavior.
◦  Stabilize repeatability and triage known incompatibilities.
9. Phase 8 — Release/CI hardening
◦  Add CI matrix (lint/type/tests + interop smoke), packaging/release flow, changelog/version policy.

Recommended execution slices
•  Slice A (MVP fast path): Phase 1 → 2 → 4
•  Slice B (network parity): Phase 3 + multi-peer integration tests
•  Slice C (cross-language confidence): Phase 6 → 7
•  Slice D (publish-ready): Phase 8
