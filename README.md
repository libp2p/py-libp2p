# Py-IPFS-Lite

A lightweight, Python-based implementation of IPFS Lite, built on top of `py-libp2p`. Py-IPFS-Lite allows you to share and fetch files over the IPFS network natively in Python without needing the full weight of a complete IPFS node.

## Features

- **Daemon Architecture**: Run a background node that allows you to seamlessly share multiple files concurrently.
- **Bitswap Protocol**: Full compatibility with the standard IPFS Bitswap protocol for block exchange.
- **CID Consistency**: Content Identifiers (CIDv1) generated by this project match standard Go IPFS / Kubo byte-for-byte.
- **Interoperability**: Readily fetch files from standard Kubo nodes or serve files to them.
- **Modern Python**: Built with asynchronous I/O using `trio` for highly concurrent networking.

## Installation

This project uses `uv` for modern, fast Python package management.

1. Ensure you have `uv` and Python 3.12+ installed.
2. Clone the repository and install dependencies:

```bash
uv sync
```

## Usage: The Daemon Workflow

Py-IPFS-Lite is designed to run via a **daemon architecture**, similar to standard IPFS tools. You start the daemon process once, which initializes the network stack, and then use the CLI from another terminal to send it commands (like adding or fetching files).

### 1. Start the Daemon
Run the following command and leave it running in your terminal:
```bash
uv run python py_ipfs_lite/main.py daemon
```
This will start the node, initialize Bitswap, and open a local API on `127.0.0.1:5002`.

### 2. Add a File
In a **new terminal window**, add a file to the network:
```bash
uv run python py_ipfs_lite/main.py add /path/to/your/file.txt
```
**Output Example:**
```
Added file successfully! CID: bafkreihca6cic4m7e55crddyhznnypcmzbqvy2zkem2bdve4wgfyoy7jli
Provider Peer ID: 12D3KooWN4SKSLrzg3jVPkPKnhWgikAXv75QrSyAEZNJTpiz9dba
Provide the following address to peers:
  /ip4/127.0.0.1/tcp/64554/p2p/12D3KooWN4SKSLrzg3jVPkPKnhWgikAXv75QrSyAEZNJTpiz9dba
```
You can repeat this command to add as many files as you want!

### 3. Fetch a File
To fetch a file from another peer, use the `get` command. Provide the CID and the multiaddress of the provider:
```bash
uv run python py_ipfs_lite/main.py get <CID> --provider <MULTIADDRESS> --out downloaded_file.txt
```
*(If `--out` is omitted, the file contents will be printed directly to the console).*

## Advanced Configuration

You can customize the node's behavior using global flags on the `daemon` command:

- **Custom Ports**: Define exactly which TCP port the node should listen on for peer connections:
  ```bash
  uv run python py_ipfs_lite/main.py daemon --port 4001
  ```
- **Deterministic Peer IDs**: Use a seed to ensure your node generates the exact same Peer ID across restarts:
  ```bash
  uv run python py_ipfs_lite/main.py daemon --seed "my-secret-seed"
  ```

## Interoperability with Kubo (Standard IPFS)

Py-IPFS-Lite is fully compatible with standard Kubo nodes via the Bitswap protocol.

### Scenario A: Fetching a file FROM Kubo USING Py-IPFS-Lite
1. On your Kubo node, add your file:
   ```bash
   ipfs add my_file.txt
   ```
2. Get your Kubo node's multiaddress:
   ```bash
   ipfs id
   ```
3. Using Py-IPFS-Lite, fetch the file by passing the CID and the Kubo node's multiaddress:
   ```bash
   uv run python py_ipfs_lite/main.py get <CID> --provider <KUBO_MULTIADDRESS> --out file.txt
   ```

### Scenario B: Fetching a file FROM Py-IPFS-Lite USING Kubo
1. Start your Py-IPFS-Lite daemon:
   ```bash
   uv run python py_ipfs_lite/main.py daemon
   ```
2. Add a file to your Python node:
   ```bash
   uv run python py_ipfs_lite/main.py add shared_file.txt
   ```
   *(Copy the Multiaddress and CID output by Py-IPFS-Lite).*
3. On your Kubo node, manually connect to the Py-IPFS-Lite node to establish the swarm connection:
   ```bash
   ipfs swarm connect <PY_IPFS_LITE_MULTIADDRESS>
   ```
4. Fetch the file using Kubo:
   ```bash
   ipfs get <CID>
   ```

## Interop Testing Suite

To strictly ensure that compatibility remains unbroken, this repository includes an interoperability test suite that spins up a lightweight Go peer and tests bidirectional 10MB and 1MB file sharing against the Python implementation.

To run the suite:
```bash
./interop/run_interop.sh
```
This requires Go to be installed on your system. It guarantees that CID calculation and Bitswap block exchanges work perfectly in both directions.
