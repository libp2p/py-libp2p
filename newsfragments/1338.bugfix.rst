Fixed a KadDHT vulnerability where a signed ``PeerRecord`` was accepted even when the envelope's signer identity did not match the ``peer_id`` claimed inside the record, allowing an attacker to certify their own addresses for an arbitrary victim peer ID (certified-address-book poisoning). ``maybe_consume_signed_record`` now rejects any record whose signer key does not derive to ``record.peer_id`` on both the ``senderRecord`` and ``signedRecord`` paths.
