Enforce IP subnet diversity in Kademlia k-buckets (issue #1383): a new peer is rejected if its globally-routable ``/24`` (IPv4) or ``/48`` (IPv6) subnet already holds ``MAX_PEERS_PER_SUBNET`` (default 2) peers in that bucket, raising the cost of eclipse attacks that grind peer IDs from a single subnet. Loopback, private, CGNAT, link-local, DNS-named, and relayed (``p2p-circuit``) peers are exempt; set ``MAX_PEERS_PER_SUBNET`` to 0 to disable.
